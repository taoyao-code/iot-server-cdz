# 项目进度与缺失清单（IoT 充电桩平台）

更新时间：2025-09-23

## 已完成（核心纵切）

- 接入与基础设施：HTTP/TCP、配置(viper)、日志(zap+lumberjack)、指标(Prometheus)、Mux 多协议首帧初判
- 协议：AP3000 最小闭环（20/21/22/12/82/03/06 解析/处理/路由），**BKV 核心功能完成**
- 会话：心跳维持在线判定、绑定设备到连接；多信号骨架（TCP 断开/ACK 超时），加权策略（可配）
- 持久化：PG 连接池与首批迁移（devices/ports/orders/cmd_log/outbound_queue）
- 出站：outbound worker（冷启扫描、ACK 超时退避、重试/置 dead、dead 清理、按协议编码、节流）
- 管理接口：健康/就绪/指标；第三方验签回环；OpenAPI 草案
- 文档与 CI：实施/架构/计划文档体系，OpenAPI 草案，基础 CI

## 协议实现状态总览

### 已完成协议

- **BKV协议（100%完成）** ✅ - `internal/protocol/bkv/`：
  - [x] 控制/参数：完整字段解析、参数写入→回读校验闭环
  - [x] 结束原因映射：统一枚举与错误码字典、智能状态推导
  - [x] 异常事件上报：完整异常信息处理
  - [x] 参数查询：设备参数解析与记录
  - [x] 刷卡充电：完整业务流程实现（余额验证、订单创建）
  - [x] OTA升级：固件升级流程处理

- **GN协议（组网设备）（100%完成）** ✅ - `internal/protocol/gn/`：
  - [x] 帧层解析：fcfe/fcff格式完整支持 - `wire.go`
  - [x] 命令路由：8种命令完整实现 - `router.go`
  - [x] 业务处理：心跳、状态、控制、参数、异常等 - `business.go`
  - [x] 可靠性保障：出站队列、ACK、重试、冷启动恢复 - `worker.go`
  - [x] 数据存储：完整的数据模型和仓储接口 - `internal/storage/gn/`
  - [x] 测试覆盖：完整的单元测试和集成测试

> **协议说明**：BKV和GN都支持组网设备，BKV更侧重于业务层处理，GN更侧重于通信层实现。两个协议可以并存，支持不同厂商的设备接入。
